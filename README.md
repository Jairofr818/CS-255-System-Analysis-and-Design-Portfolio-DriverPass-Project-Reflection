# CS-255-System-Analysis-and-Design-Portfolio-DriverPass-Project-Reflection
DriverPass project entailed developing a web application for Liam who is the owner of DriverPass. Customers could take online classes and practice exams, buy training bundles, and book driving lessons. Employees were able to manage accounts, appointments, drivers, vehicles, reporting, and DMV updating.

It was easy for me to determine the system users and the system requirements. I successfully specified what the customers, secretaries, instructors, managers, and IT staff needed to do inside the system.

The area that I would improve is the set of UML diagrams which I would like to make more readable by reducing lines and labels.

I determined the needs of the users by analyzing the interview with the client and transforming his requests into system requirements and UML diagrams. It is very important to think about the user needs as the system should resolve the client's problem and be user-friendly.

When creating the software, I determine the users, their needs, processes and constraints. In the future, I will apply this approach based on interviews, requirement list, user stories, and UML diagrams.
